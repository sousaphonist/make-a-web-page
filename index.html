Important Concepts

Lesson 1: The Basics of the Web and HTML

How the Internet Works

The internet is a series of interconnected computers. When you attempt to open a webpage, your computer sends an HTTP Request to a server.

The server then finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen.

HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web.

HTML documents contain text content which describes "what you see" and markup which describes "how it looks".

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags.

A tag takes the form < tag > content < / tag >. The "/" denotes that it is a close-tag. Without it, the tag would run to the end of the doctument. This is where most code errors occur, as I shall explain below.

Computers are Stupid

As humans, we can often see someone's little mistake, understand what they meant it to be, and take it as such. Computers, however, cannot do this. They are rigidly logic-driven machines, and can only see what is right in front of them. This is where most errors in code appear - due simply to a missing bracket or close tag.

Inline vs Block Elements

HTML elements are either inline or block.

Block elements form an "invisible box" around the content inside of them, and are useful to break up text into paragragraphs or for giving elements a more visually appealing layout.

Inline elements effect how the text itself is presented. For example, they can make text bold or italicized.

Lesson 2: Creating a Structured Document with HTML

Developer Tools

Everything displayed on a webpage, from titles and paragraphs to pictures and videos, are contained within their own little box. Developer Tools, often simply referred to as Dev Tools, allow you see all the boxes that make up a given webpage, as well as all the boders and/or spacing around them. Additionally, Dev Tools may be used to view the page's CSS (more on that later).

You can then play around with the code of the website - adding, deleting, or simply modifying whatever you want. These changes are only temporary, only you can see them and only for as long as you stay on the webpage; you can also revert the page back to normal simply by refreshing the page.

Opening up and the exact form of the Dev Tools is different for each browser. In general, you should be able to find them in the same place as Settings or Options and looking for Tools. Of course, when did doubt, just Google it.

The "tree-like structure" of HTML

HTML is said to have a "Tree-Like Structure" in that practically every webpage is set up as boxes within boxes within boxes. Think of a webpage as like the quest of the "Lord of the Rings" book trilogy - each book would be it's own box, each chapter would be a sub-box, and each paragraph would be a sub-sub-box.

HTML and Indented Boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is. This can be useful in trying to track down the sources of errors, as well as making sure all necessary paragraph tags are in place.

Text Editors

When writing code, programmers use special text editors. Some common ones include Sublime Text, Scratchpad, and Codepen. These can make programming easier, by automatically generating the necessary close-tags and/or providing a real-time result of the code.

Lesson 3: Adding CSS Style to HTML Structure

Avoiding Repetition

Avoiding repetition is important for a variety of reasons.

Consistency: Often, there will be multiple boxes within the HTML which should all look the same. If you add the style choices to each one individually, then every time you change your mind about how you want it to look, you'd have to remember to change each and every box by hand. Obviously, this'd get very tiresome, so it's better to just have once place where you can change the code once and have it affect every box.

Avoiding Errors: The more characters are needed to produce your code, the more likely it is you're going to make a mistake. As I mentioned before, even the smallest mis-type can result in a page-breaking error, so it's best to streamline your code as much as possible.
